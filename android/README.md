# Android build (Health Tracker)

The native Android project lives at **[`../src-tauri/gen/android`](../src-tauri/gen/android)** (generated by Tauri). This folder only documents how to build and install the APK.

## Prerequisites

1. [Android Studio](https://developer.android.com/studio) with SDK Platform, Platform-Tools, Build-Tools, Command-line Tools, and **NDK (Side by side)**.
2. Environment variables (or run the helper script):
   - `ANDROID_HOME` — e.g. `%LOCALAPPDATA%\Android\Sdk`
   - `NDK_HOME` — e.g. `%ANDROID_HOME%\ndk\<version>`
   - `JAVA_HOME` — e.g. `C:\Program Files\Android\Android Studio\jbr`
3. Rust Android targets:

```powershell
rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android
```

From the repo root, load env vars automatically:

```powershell
. .\scripts\android-env.ps1
```

## Windows: "Failed to create a symbolic link"

`npm run android:dev` uses a **copy-based workaround** so you do not need Developer Mode. It starts Vite, builds the Rust library, copies `libtauri_app_lib.so` into `jniLibs`, and runs Gradle `installArm64Debug`.

To use the stock Tauri dev server (Rust hot-reload via `tauri android dev`), enable **Settings → System → For developers → Developer Mode**, then run `npm run android:dev:tauri`.

For a **physical phone** on the same Wi‑Fi (not the emulator), run:

```powershell
npm run android:dev
```

The script auto-detects a real device and uses your PC's LAN IP instead of `10.0.2.2` (which only works in the Android emulator). If you see `Failed to request http://10.0.2.2:1420/`, re-run `npm run android:dev` with the phone connected and Vite running.

To force emulator mode: `.\scripts\android-dev.ps1 -Emulator`

## Commands

| Command | Description |
|---------|-------------|
| `npm run android:dev` | Dev on emulator/USB (Windows: copies `.so` instead of symlinks) |
| `npm run android:dev:tauri` | Stock `tauri android dev` (requires Windows **Developer Mode**) |
| `npm run android:build` | arm64 debug APK (Windows-friendly; copies `.so` instead of symlinks) |
| `npm run android:build:tauri` | Full `tauri android build --apk` (needs Windows Developer Mode for symlinks) |

First-time setup (if `gen/android` is missing):

```powershell
. .\scripts\android-env.ps1
npm run tauri android init
npm run android:dev
```

## Install APK on a phone

After `npm run android:build`, the installable APK is:

`src-tauri/gen/android/app/build/outputs/apk/arm64/debug/app-arm64-debug.apk`

Other variants may appear under `src-tauri/gen/android/app/build/outputs/apk/`.

```powershell
adb install -r path\to\app-universal-release.apk
```

Or copy the APK to the device and allow installation from unknown sources.

## Data

Android uses app-private storage (not the Windows `%APPDATA%` path). Use **Settings → Export backup** to move data between desktop and phone.
